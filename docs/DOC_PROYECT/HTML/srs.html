Especificación de Requerimientos

  1. Introduccion

    El presente documento contiene el propósito por el cual se realizará el proyecto “Software a la medida del sector empresarial”, el alcancé que tendrá, las definiciones acerca de palabras poco comprensibles y las referencias de donde se sacó dicha información También abarca las perspectivas y funciones del producto, además de las características de usuario y por último lleva los requerimientos específicos, que son: Los requerimientos funcionales, los requerimientos no funcionales y los requerimientos de interfaz de usuario

    1. Proposito

      propósito de este proyecto es, elaborar un aplicativo web para el mejoramiento de las condiciones de la I.E.C.R “Institución Educativa Castillo del Rey” al tener organizada la información y así disminuir el tiempo implementado en el trabajo al poseer más accesibilidad y llevar de una manera ordenada la información relevante de la Institución


    2. Alcance

      Este proyecto, consta con 7 fases las cuales son:

      • Inducción
      • Análisis
      • Desarrollo
      • Diseño
      • Identificación
      • Implantación

        • En la cual nos comprometemos a realizar 6:

      • Inducción
      • Análisis
      • Desarrollo
      • Diseño
      • Identificación

        • Nos comprometemos a realizar un aplicativo web para el control y manejo de:

      • Registro e informes de asistencias.
      • Registros e informes de notas
      • Pago de mensualidades
      • Registro de usuario

    3. Definiciones, Acrónimos, y Abreviaturas

      • Usuario: Personal que usará el sistema de información
      • Base de datos: Es una recopilación organizada de información o datos estructurados, que normalmente se almacena de forma electrónica en un sistema informático.
      • Login: Es el proceso que controla el acceso individual a un sistema informático mediante la identificación del usuario utilizando credenciales provistas por el usuario.
      • Software: Son los programas informáticos que hacen posible la ejecución de tareas específicas dentro de un computador.
      • Diseño: Es el proceso de diseño para la planificación de una solución de software
      • Desarrollo: Conjunto de actividades informáticas dedicadas al proceso de creación, diseño, despliegue y compatibilidad de software
      • Implementación: Es la ejecución o puesta en marcha de una idea programada
      • Calidad: Capacidad que posee un objeto para satisfacer necesidades implícitas o explícitas según un parámetro
      • Hardware: Conjunto de elementos físicos o materiales que constituyen una computadora o un sistema informático.
      • Aplicativo: Programa informático diseñado como una herramienta para realizar operaciones o funciones específicas.
      • Información: Conjunto de datos que tiene un significado, y cuyo objetivo es ampliar conocimientos, informar o aportar ideas sobre un tema determinado

  2. Descripción General

    1. Perspectivas del Producto

      Las perspectivas del producto son la elaboración de un aplicativo web, el cual va a estar en capacidad de llevar el registro de los usuarios, la asistencia de cada alumno, las notas de dichos estudiantes y el control de pago de mensualidades


    2. Funciones del Producto

      Las funciones con las que contará el sistema son:
      1. Registro de usuarios:
        • Usuario Administrativo
        • Usuario para docentes
        • Usuario para alumnos/padres de familia
      2. Iniciar sesión con el documento de identidad y contraseña
      3. Menú principal, con diferentes funciones, dependiendo del rol del usuario
      4. Registro de docente, que a su vez permitirá inhabilitarlos cuando ya no haga parte de la institución
      5. Registro de asistencia de cada alumno
      6. Registro de notas
      7. Realización de boletines

    3. Características de Usuario

      El sistema cuenta con tres tipos de usuarios:

      • El primero usuario se le atribuye al personal directivo, en este caso a la directora, con un nivel de educación superior, tener por lo menos, un nivel intermedio del uso de la informática.

      Las funciones que cumple este usuario son:
      • Registrar a los docentes y a su vez poder inhabilitarlos si ya no hacen parte de la institución.
      • Llevar el control de los pagos de mensualidades de cada alumno
      • El siguiente corresponde a los docentes, quienes deben tener un nivel de educación
        superior y conocimientos básicos acerca de computación
      Las funciones que tienen, son:
      • Registrar la asistencia de cada alumno
      • Ingresar las notas a los estudiantes
      • Realizar los boletines correspondientes de cada niño
      • El ultimo usuario son los alumnos/padres de familia, los cuales quizá no tengan el conocimiento acerca de cómo usarlo.
      • Este usuario tendrá la posibilidad de ver su progreso/nivel académico

    4. Restricciones

      • Recurso presupuestal insuficiente para la implantación.
      • No aceptación de la solución tecnológica por parte del cliente.
      • No satisfacer completamente las necesidades específicas del cliente.
      • Fallas en el control de calidad en el desarrollo del proyecto.
      • Pérdida o daño en los recursos tecnológicos adquiridos para el proyecto.
      • Pérdida o daño en los dispositivos de almacenamiento de información.
      • Daños o problemas con la conectividad.

    5. Atención y Dependencias

      Las áreas que se verían afectada por el software podrían ser:
      • Administrativas
      • Ambiental
